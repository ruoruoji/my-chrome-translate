## v1.0.2 (2026-02-04)

本版本重点修复了在“选中文本自动弹出”场景下，播放和复制功能可能失效或需要等待翻译完成的问题。同时，我们增强了浮层按钮在特定页面（如微信公众号文章）上的点击稳定性，确保交互体验更加流畅可靠。

### ✨ 功能增强

- **播放/复制在“选中文本自动弹出”场景下立即可用**
  - **播放、复制兜底选区文本**：当您通过划词选择文本触发自动弹窗后，无论翻译是否完成或成功，点击“播放”或“复制”按钮都会立即作用于当前页面上实际选中的文本。这解决了此前需要等待网络请求返回才能操作的延迟问题，确保了功能的即时响应。
  - **新增 `getCurrentSelectionText()`**：为了实现上述改进，我们在内容脚本中引入了新的工具函数 `getCurrentSelectionText()`。该函数能够在插件内部数据状态（`currentData`）尚未更新时，可靠地获取用户当前的选区文本，作为播放和复制功能的即时数据源。

- **宿主元素 `pointerEvents: auto`，增强按钮点击稳定性**
  - 我们将浮层宿主元素的 `pointer-events` CSS 属性从 `none` 调整为 `auto`。此项改动主要解决了在某些页面结构复杂的网站（例如微信公众号文章页）中，浮层按钮的点击事件可能被意外拦截或“穿透”的问题，显著提升了插件的兼容性和交互稳定性。

### ✅ 保留的既有逻辑

本次更新专注于体验优化，未影响现有核心逻辑，包括：

-   **仅最新请求更新**：当您快速连续触发翻译时，只有最新一次的请求结果会更新到浮层上，避免了旧结果覆盖新结果的混乱情况。
-   **双击与划词触发**：双击单词和划词自动弹出的两种触发机制保持不变。
-   **全大写缩略词朗读**：对于如 "API"、"NASA" 等全大写的缩略词，TTS 引擎将继续采用逐字母朗读的方式，以符合通常的发音习惯。
